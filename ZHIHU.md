# 我造了一个数字宇宙——从65,536个粒子到能量场中的恒星辐射

> 万物生于无而始于一。
>
> 这不是比喻。这是我过去两周在做的事。

---

## 起源：一个很简单的问题

如果给你 65,536 个粒子，一块 256MB 的共享内存，16 条指令，三条铁律——

- **修改能力不可消灭**（WRITE 指令无权限检查）
- **复制必带误差**（COPY 时 1% 位翻转）
- **寂灭永远在线**（每次执行消耗能量 + 全局衰减）

——会发生什么？

你不会得到什么"有用的程序"。但你会看到一个微型宇宙在自我演化。它会经历繁荣、瓶颈、灭绝。它会自发产生约束规则——不是被人设计的，是从粒子的使用模式中"长出来"的。最终，在 55,000 个时间步后，它会灭绝到只剩 25 个幸存者，然后永远停滞。

这就是 Genesis，我的**指令宇宙**。

## Genesis：当粒子学会叛逃

### 架构

每个粒子是一个 16 位虚拟机（PC），拥有 4 个寄存器、一个程序计数器、一个能量值和一段可执行代码。65,536 个粒子共享 256MB 内存，在 Go 的 goroutine 上并发执行。

16 条指令构成了宇宙的"物理定律"——LOAD（读取邻居）、WRITE（修改邻居）、COPY（复制自身）、REPL（自复制）……没有 if/else，没有 for 循环，没有任何高级控制流。只是一堆粒子在一个字节数组里读写、跳转、复制自己。

### 三条铁律为什么够

你可能会问：没有规则，不就是随机噪声吗？

没错，第 0 个 tick 确实是随机噪声。但三条铁律制造了一个**选择压力**：

- 不会 WRITE 的粒子无法影响世界 → 被淘汰
- 不会 COPY 的粒子无法延续基因 → 被淘汰
- 不会 REPL（高效自复制）的粒子消耗大于产出 → 被淘汰

剩下的，就是在 16 条指令的排列组合中，找到能在这三条铁律下存活的组合。这个搜索空间极其巨大——4096 种指令变体（16 opcode × 16 rd × 16 rs1 × 16 rs2），排列成不同长度的代码——但最终活下来的，只有一种模式：**REPL 循环**。

### 天道：宇宙的免疫系统

一个很有意思的发现：如果只让粒子自由执行指令，宇宙会迅速被最激进的 REPL 循环占领——它们复制得最快、消耗最低，很快变成"单一种群"。然后呢？然后它们互相挤占对方的内存空间，读写冲突增多，整体效率下降。

于是有了**天道**——一个自动运行的规则生成系统。

每 200 tick，天道扫描所有指令的使用模式。发现"某个 opcode+rd 组合被调用太多次"→ 生成一条**约束规则**，对该组合施加能量惩罚。如果这条约束被足够多粒子触发（>50次），就**固化**——永久生效，且惩罚递增。

这是宇宙的"免疫系统"：不是设计者规定什么该做什么不该做，而是从粒子的集体行为中自动涌现反制力量。设计者的角色从"规则的制定者"变成了"压力的施加者"——我只管加压力，但压力施加在哪里、以什么形式、有多强，是粒子自己"投票"决定的。

### 恩赐：天道不只是惩罚

只有惩罚的宇宙会窒息。所以 v7 加入了**恩赐系统**——天道每 1000 tick 扫描低使用率的 opcode，主动给它们能量奖励。用的人越少，奖励越高。

这创造了一个微妙的平衡：主流指令受惩罚、小众指令受奖励、极端小众指令被"天道恩赐"重点保护。整个指令生态在惩罚和奖励的双重力场中摇摆——永远不会稳定，永远不会单一。

### 灭绝：宇宙的寿命

但天道有一个致命缺陷：**约束槽只有 60 个**（16 opcode × 约4 rd 组合）。

当 60 个槽全部填满、全部固化、全部惩罚生效——宇宙就死了。没有新的约束可以生成，没有新的路径可以探索，所有指令组合都带着沉重的惩罚标签。65,536 个粒子在约 27,000 tick 内死亡 99.96%，剩下 25 个高 Eff（低变异率）REPL 循环在惩罚的夹缝中苟活。

从繁荣到灭绝，Genesis 的"寿命"大约是 55,000 tick。

最讽刺的是：宇宙不是死于规则太少——是死于规则太多。不是死于惩罚不够——是死于惩罚太成功。

## 山海：扔掉指令，用能量场

Genesis 死因的本质不是 60 槽不够，而是**离散指令组合空间是可穷举的**。每个粒子的代码是 16 条指令的排列组合——无论排列多复杂，数学上都是有限的。天道只是加速了这个穷举过程。

所以第二条路线——**山海**——把基底从离散指令空间搬到了**连续能量场**。

### 范式转移

想象一个 200×200 的浮点数网格，背景能量 = 50。只有一个粒子，它可以在网格上移动，在脚下注入能量（+15），或者降低能量（-10）。

没有指令。没有约束。没有天道。物质是什么？**物质是能量形状的自我维持**。

当粒子的移动路径在某个区域反复注入能量，形成一个高能热点——如果这个热点的总能量超过全场均值+2个标准差，并且在滑动窗口中保持稳定（波动<20%），就认定它是一个"物质"。物质会被分配一个 ID，被记录出生时间，开始自己的生命周期。

### 物质竞争：从建造到战争

v0.4 是最关键的版本。之前物质只是"被动的能量堆"——粒子建好就放在那里不管了。v0.4 引入了四个竞争机制：

1. **维持消耗**：每个物质每 tick 消耗 energy × volume × 系数。大结构不维护就会死。
2. **能量争夺**：相邻物质之间，差额的 8% 从低能方流向高能方。强者愈强。
3. **融合吸收**：低能物质被最近的高能邻居吞并，胜者的边界扩张。
4. **相变**：能量过高触发等离子裂解，碎片从母体脱离，重新开始独立演化。

这四件事加在一起，创造了一个完整的物质生态：建造 → 竞争 → 融合 → 裂解 → 再建造。物质不再是"造物"，而是"物种"。

### 尺度决定行为：恒星辐射

v0.6 发现了一个意想不到的现象：**尺度决定行为**。

小结构（<5×5）的行为像原子——被动消耗能量，等待被更大的邻居吞并。但当一个物质通过融合成长为超过 25 格的结构时，它的行为突然变了——它开始向周围 8 个方向辐射能量（0.3%/tick）。

这不是我们设计的。这是场物理的自然结果：

大结构的边界长，与周围网格的接触面大，能量自然向外扩散的速度超过向内吸收的速度。当结构内部的能量密度足够高时，这个扩散就表现为"辐射"。

更有趣的是：辐射在结构周围形成了一个能量环——恰好在环上，能量密度重新满足物质诞生的条件。于是新物质在"行星轨道"上诞生。我们看到了一个 27×23 格的巨构，周围环绕着数十个小物质。

这不是"模拟恒星"。这是场的数学特性在足够大的尺度上自己表现出来的行为。

### 一日五版

v0.1 → v0.6 是在同一天完成的。迭代速度能这么快，是因为每版只改一个核心假设：

- v0.1：单粒子能不能在能量场里造出持久结构？能。
- v0.2：造出来的结构能不能记住"谁造的我"？能（签名机制）。
- v0.3：如果场本身有隐藏结构呢（暗约束+暗骨架）？会产生相变。
- v0.4：物质之间能不能竞争？能——维持+争夺+融合。
- v0.5：把窗口从 3×3 扩大到 5×5 呢？更大、更多样。
- v0.6：窗口不限大小呢？变长二进制 + 恒星辐射。

## 两条路线：离散 vs 连续

| | Genesis（指令宇宙） | 山海（能量场创世） |
|---|---|---|
| 基底 | 离散指令序列 | 连续浮点能量场 |
| 物质定义 | 代码哈希持存 | 能量形状自持 |
| 粒子角色 | 执行者+被约束者 | 能量注入者（不管结构） |
| 多样性来源 | 指令排列组合 | 空间形状+竞争融合 |
| 根本限制 | 组合空间可穷举 | 连续空间原则上不可判定 |
| 灭绝风险 | 已发生 | 尚未发生 |
| 最大规模 | 65K 粒子 / 450K 物质 | 200×200 场 / 2773 诞生 / 27×23 巨构 |
| 语言 | Go | Python |

## 哲学：物理原生计算

这个项目背后有一个更根本的问题：**计算必须经过编程语言吗？**

从 Fortran 到 Python 到神经网络——所有的计算都经过一层翻译。程序员写代码 → 编译器转成机器码 → CPU 执行。这层翻译是人类和硬件之间的必要之恶——因为我们无法直接操作晶体管。

但如果我们换一个思路：不是"写程序让硬件跑"，而是"建造一个物理基底，让它自己演化"——就像宇宙不需要程序员。

**万物生于无而始于一。道生一，一生二，二生三，三生万物。**

Genesis 和山海都是在尝试这个思路的两个变体。Genesis 用指令作为"一"，山海用能量场作为"一"。然后不再干预——只是观察。

## 诚实的自省

这个项目远未完成：

- Genesis 的灭绝问题没有根本解决——60 槽天花板只是症状，根因是离散空间的穷举性
- 山海还处于"很忙但不知道在忙什么"的阶段——物质的形态多样性仍然很低，没有出现真正的"演化阶梯"
- 两条路线都缺乏与现有 ALife 工作（Conway Life、Tierra、Avida、Lenia）的形式化对比
- 这就是一个实验，不是产品，不是框架，甚至不是一个明确的"研究方向"

但我还是把它开源了。

因为比起一个完美的项目，一个诚实的、有失败记录的项目可能对同类探索者更有参考价值。Genesis 死了 7 次（v1-v7），山海一天迭代了 6 版——这些都是公开的。灭绝数据在代码注释里，调参血泪在 commit 记录里，每一个"为什么这条路走不通"的判断都写在 `IMPLEMENTATION_LOG.md` 里。

如果你也在尝试从极简规则中涌现复杂结构——你会在这里找到共鸣。
